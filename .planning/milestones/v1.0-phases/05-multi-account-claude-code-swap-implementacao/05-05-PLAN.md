<objective>
Adicionar campo `claudeConfigDir` opcional ao config schema consumido por `executeClaudeLocal` em `packages/adapters/claude-local/src/server/execute.ts`, e fazer o wiring para `env.CLAUDE_CONFIG_DIR` antes do spawn. Conforme Finding 2 da Phase 4, `CLAUDE_CONFIG_DIR` já está em `includeRuntimeKeys` (linha 253) e propaga via process env; este plano formaliza a passagem por config (em vez de só por env do processo) para que o caller (heartbeat em 05-06) controle qual conta o spawn usa via objeto config.

Purpose: Habilita o caller (heartbeat) a injetar dinamicamente o diretório de credenciais selecionado pelo `selectActiveAccount` em cada spawn, sem precisar mexer em process.env (que seria global e não thread-safe entre múltiplos agentes concorrentes).

Output: 1 modificação cirúrgica em execute.ts (~5-10 linhas) + 1 test case adicionando.
</objective>
