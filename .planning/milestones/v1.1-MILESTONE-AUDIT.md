## 1. Resumo Executivo

Milestone v1.1 entregou internacionalização completa do paperclip team-shared para português do Brasil — UI inteira, mensagens dos agentes, e system prompts dos modelos LLM honram a preferência de idioma do usuário. 26/26 requisitos satisfeitos a nível de código; 16 UATs empíricos (browser real + agentes reais) ficam pendentes para o operador validar conforme uso natural, seguindo o precedente `complete-with-pending-UAT` estabelecido nas Fases 3-6 do milestone v1.0.

**Escopo entregue:**
- Toggle pt-BR/en-US em `instance/settings` persistido em `authUsers.locale` (Better Auth schema, migration 0073)
- Infraestrutura i18n (i18next 26 + react-i18next 17, 8 namespaces, ~1700 chaves em 2 locales, missing-keys CI gate)
- UI core traduzida: Inbox, Projects, Settings, Nav (645/645 testes UI verdes ao final da Fase 8)
- UI admin/auth/sistêmicas: 11 admin pages + 4 auth flows + 32 server error codes mapeados (685/685 testes verdes ao final Fase 9)
- Mensagens dos agentes: AgentDetail (4248 LOC), RunTranscript (1526 LOC), LiveUpdatesProvider tRef pattern (706/706 testes verdes Fase 10)
- System prompts: resolveRunOwnerLocale + buildLanguageDirectiveBlock + 4 SKILL.pt-BR.md variants + bundle cache locale-aware
- Activity log com schema `actionKey + paramsJson` (migration 0074) backwards-compat para entries legados

**Tempo total:** ~462 minutos de executor wall time (5 fases, 19 planos, ~85 commits).

## 4. Integração Cross-Fase

**Fluxos E2E verificados (5 wired):**

1. **Locale roundtrip** (Fase 7 → Fases 8-11): UI mutation → PATCH /api/auth/profile → DB → req.locale middleware → next session hydrates
2. **Activity log roundtrip** (Fase 8-05 → consumers): server actionKey+paramsJson → DB → client t() with legacy fallback
3. **Error code roundtrip** (Fase 9-03a → 9-03b): server *WithCode → ApiError.code → translateApiError → toast/modal
4. **Agent toasts via tRef** (Fase 10-03): tRef pattern evita reconexão WebSocket em troca de idioma
5. **Skill variant materialization** (Fase 11-03): materializeSkillForLocale + bundleKey v2 isolation

**HIGH finding remediado mid-audit:**
- **Identificação:** integration-checker detectou que `execute.ts:537` gating `sessionId ?` impedia injeção do directive no first-spawn (apenas resume tinha fallback)
- **Fix:** commit `525368f` removeu o gating — directive sempre aplicada (no-op para en-US). 3 linhas alteradas.
- **Impacto:** AGENT-SKILL-01 agora coberta tanto em first-spawn quanto em resume sem depender de heartbeat opt-in via exportFiles({locale}).
