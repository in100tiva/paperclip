<objective>
Wave 1 / Foundation. Land the persistence + type contracts for user locale:

1. Add `locale` column to `user` table (Drizzle schema + migration).
2. Register `additionalFields.locale` in Better Auth so `session.user.locale` is typed and populated.
3. Extend Zod validators (`localeSchema` enum, `currentUserProfileSchema.locale`, `updateCurrentUserProfileSchema` with all-optional fields + `.refine()`).

After this plan, the **server-side type contract** for locale exists end-to-end: DB → Better Auth session → shared Zod types. Plans 04 (server route + middleware) and 05 (UI) build directly on top.

Purpose: Without this contract, no other plan can land — the route can't accept `locale`, the middleware can't read it from session, and the UI can't infer the field on `CurrentUserProfile`. This is the structural unblock.

Output: 1 new migration file, 4 modified TypeScript files, all `pnpm tsc --noEmit` clean.
</objective>

```typescript
export const localeSchema = z.enum(["pt-BR", "en-US"]);

export const currentUserProfileSchema = z.object({
  id: z.string().min(1),
  email: z.string().email().nullable(),
  name: z.string().min(1).max(120).nullable(),
  image: profileImageSchema.nullable(),
  locale: localeSchema,  // NEW — DB has NOT NULL DEFAULT, always returned
});
export type CurrentUserProfile = z.infer<typeof currentUserProfileSchema>;

// MODIFIED: all fields optional + .refine()
export const updateCurrentUserProfileSchema = z
  .object({
    name: z.string().trim().min(1).max(120).optional(),  // was required
    image: z
      .union([profileImageSchema, z.literal(""), z.null()])
      .optional()
      .transform((value) => (value === "" ? null : value)),
    locale: localeSchema.optional(),  // NEW
  })
  .refine(
    (data) => data.name !== undefined || data.image !== undefined || data.locale !== undefined,
    { message: "At least one field (name, image, or locale) must be provided" },
  );
export type UpdateCurrentUserProfile = z.infer<typeof updateCurrentUserProfileSchema>;
```
