<known_pitfalls>
From `.planning/research/PITFALLS.md`:
- Paperclip uses an `embedded-postgres` patch (see `patchedDependencies` in `package.json`). pnpm install must apply this patch — if the patch fails, embedded-pg won't start.
- Paperclip's `dev-runner.ts` spawns server + UI in parallel. On Windows, sometimes ports 3000/5173 (or whatever paperclip uses) are blocked by Windows Defender Firewall on first run — user may need to approve a firewall prompt.
- This is the FIRST run of paperclip on this machine — embedded-pg downloads its binary tarball on first init. Allow extra time on the smoke test (network-dependent).

Failure modes that would invalidate the baseline (must be diagnosed, NOT papered over):
- `pnpm install` failing on a `patches/embedded-postgres+*.patch` apply error → indicates Windows + pnpm patch handling problem.
- `pnpm dev` crashing immediately with "embedded-postgres failed to download" → network/proxy issue OR paperclip-side bug we inherited.
- Server starting but UI failing to compile (Vite errors) → Node version mismatch (paperclip needs >=20, fails on 18).
</known_pitfalls>
</context>
