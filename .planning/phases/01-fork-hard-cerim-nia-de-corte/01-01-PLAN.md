<clone_strategy>
**Decision: Add paperclip as a temporary remote, fetch master, then populate the working tree using `git read-tree` + `git checkout-index`. Justification:**

- `git clone` cannot target a non-empty directory with existing `.git/`.
- Cloning to temp + copying files works but loses the SHA traceability that the git history would naturally record.
- `git pull --allow-unrelated-histories` would create a merge commit and immediately conflict on `package.json` and `README.md` (since we already have planning docs but no source — fewer file conflicts than expected, but the merge commit pollutes the cut ceremony intent).
- **Chosen approach** keeps the existing `.planning/` commits intact, brings in paperclip's tree as a fresh import (one commit), and lets us record the exact upstream SHA in both git history and `UPSTREAM_REFERENCE.md`. The temporary remote is deleted before the final commit, so `git remote -v` post-ceremony lists nothing pointing to paperclipai.

Concrete commands embedded in Task 1 below.
</clone_strategy>
