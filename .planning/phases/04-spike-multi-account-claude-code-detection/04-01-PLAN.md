6 linhas obrigatórias (uma por tipo de D-05):

1. **rpm_transient** — Trigger: limite de requests/minuto excedido. Mensagem: `rate_limit_error` ou "too many requests" ou HTTP 429 sem retry-after longo. Headers: `anthropic-ratelimit-requests-remaining: 0`, `anthropic-ratelimit-requests-reset: <ISO>`, `retry-after: <seconds>`. Comportamento: transient (segundos). Cobertura: **yes** — `CLAUDE_TRANSIENT_UPSTREAM_RE` casa `rate[-\s]?limit`, `rate_limit_error`, `too\s+many\s+requests`, `\b429\b`. Regex que cobre: `CLAUDE_TRANSIENT_UPSTREAM_RE` (parse.ts:13).

2. **tpm_transient** — Trigger: limite de tokens/minuto excedido. Mensagem: `rate_limit_error` com message "tokens-per-minute exceeded" ou similar. Headers: `anthropic-ratelimit-tokens-remaining: 0`, `anthropic-ratelimit-tokens-reset: <ISO>`. Comportamento: transient (segundos a 1min). Cobertura: **partial** — regex casa `rate_limit_error` mas NÃO distingue RPM vs TPM (mesma string trigger; gap se Phase 5 quiser políticas diferentes por dimensão). Regex que cobre: `CLAUDE_TRANSIENT_UPSTREAM_RE` parcialmente (não diferencia RPM/TPM).

3. **daily_quota** — Trigger: cota diária do plano consumida. Mensagem: "usage limit reached" / "usage cap reached" / "out of extra usage". Headers: pode incluir `retry-after` apontando para dia seguinte. Comportamento: hard quota (horas até reset diário). Cobertura: **yes** — `CLAUDE_TRANSIENT_UPSTREAM_RE` casa `usage\s+limit\s+reached|usage\s+cap\s+reached|out\s+of\s+extra\s+usage`; `CLAUDE_EXTRA_USAGE_RESET_RE` extrai timestamp de "resets at X". Regex que cobre: `CLAUDE_TRANSIENT_UPSTREAM_RE` + `CLAUDE_EXTRA_USAGE_RESET_RE` (parse.ts:13-14).

4. **weekly_quota** — Trigger: cota semanal do plano consumida. Mensagem: "weekly limit reached" + "resets at <ISO>". Headers: `retry-after` apontando para próxima semana. Comportamento: hard quota (dias). Cobertura: **yes** — `CLAUDE_TRANSIENT_UPSTREAM_RE` casa `weekly\s+limit\s+reached`; reset extraído por `CLAUDE_EXTRA_USAGE_RESET_RE`. Regex que cobre: `CLAUDE_TRANSIENT_UPSTREAM_RE` + `CLAUDE_EXTRA_USAGE_RESET_RE` (parse.ts:13-14).

5. **org_tier** — Trigger: limite por organização (todos os usuários da org juntos atingiram cap). Mensagem: pode usar `overloaded_error`, `service_unavailable`, ou string explícita "organization usage limit". Headers: tipicamente sem `retry-after` claro; pode incluir `anthropic-organization-id`. Comportamento: hard quota organizacional (potencialmente longa). Cobertura: **partial** — regex casa `overloaded(?:_error)?`, `service\s+unavailable`, `\b503\b`, `\b529\b` mas NÃO tem token específico para "organization tier" — gap se Anthropic emitir mensagem distinta. Regex que cobre: `CLAUDE_TRANSIENT_UPSTREAM_RE` parcialmente (genérico, não distingue org-level).

6. **session_5h** — Trigger: limite de 5h da sessão Claude Pro/Max. Mensagem: "5-hour limit reached" / "5h limit reached" / "Claude usage limit reached" + "resets at <timestamp>". Headers: stream-JSON event ou stderr; sem header HTTP padrão (é estado de sessão, não rate-limit HTTP). Comportamento: hard quota por janela rolling de 5h. Cobertura: **yes** — `CLAUDE_TRANSIENT_UPSTREAM_RE` casa `5[-\s]?hour\s+limit\s+reached|claude\s+usage\s+limit\s+reached`; reset extraído por `CLAUDE_EXTRA_USAGE_RESET_RE`. Regex que cobre: `CLAUDE_TRANSIENT_UPSTREAM_RE` + `CLAUDE_EXTRA_USAGE_RESET_RE` (parse.ts:13-14).
