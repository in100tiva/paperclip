## Hipótese de Causa Raiz

A criação de projeto trava porque **a UI está conversando com um processo zombie** (`PID 135416`, filho de `PID 44633` que é `tsx watch` órfão de 07:13). Esse zombie:

1. Tomou as portas `:3100` (HTTP) e `:13100` (WebSocket) antes do `pnpm dev` legítimo das 09:50 nascer
2. Roda código fonte de **antes** dos 9 fixes de hoje, então não tem `idle_timeout: 1`, nem TCP `keep_alive`, nem o middleware de request timeout, nem o detector de órfão
3. Tem pool Supavisor com sockets dead-but-occupied (não detectados por causa do `idle_timeout: 5` antigo + ausência de TCP keepalive)
4. Event loop saturado por queries pendurando — não consegue nem completar handshake WebSocket

O `pnpm dev` legítimo do usuário (`PID 135245` + descendentes) está rodando mas inerte, porque sua tentativa de bind em `:3100` falha silenciosamente (ou ele detectou o registry record do zombie e adotou-o como serviço, fazendo `process.exit(0)`).

**Por que os fixes não pegaram:** o ciclo "edit → commit → testar" assumiu implicitamente que `pnpm dev` reiniciaria o processo. Não reiniciou — o zombie continuou atendendo. Cada teste do usuário foi contra binário antigo, daí a regressão aparente do problema após cada fix.
