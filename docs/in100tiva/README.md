# in100tiva — Per-Company Customizations

This directory documents customizations applied to the `in100tiva` company that are NOT generated by the `sync-agents` / `sync-skills` scripts.

The pre-existing CEO (`d64a9f21-3ad0-4ca5-b7e8-58dbefb55b75`) is intentionally excluded from `sync-agents` (see `scripts/sync-agents/mapping.ts` header comment), so its instruction bundle must be applied by hand on each fresh installation.

## Files

- **`CEO-AGENTS.md`** — Custom `AGENTS.md` for the in100tiva CEO. Replaces the default CEO bundle (`server/src/onboarding-assets/ceo/AGENTS.md`) which routes to `CTO`/`CMO`/`UXDesigner` (agents that don't exist in in100tiva). This version routes to the 4 real Heads (`planner`/`executor`/`verifier`/`user-profiler`) and adds a non-negotiable mandate to fork every task into 2+ parallel sub-issues whenever components are independent.

## How to apply on a new machine

After running the standard onboarding (`pnpm dev` + `pnpm sync-agents` + `pnpm sync-skills`):

```bash
# Copy the customized CEO bundle into your local instance
INSTANCE_DIR="$HOME/.paperclip/instances/default/companies/4b0a1c03-b502-4f28-acfd-dfd646cd5cf6/agents/d64a9f21-3ad0-4ca5-b7e8-58dbefb55b75/instructions"
cp docs/in100tiva/CEO-AGENTS.md "$INSTANCE_DIR/AGENTS.md"
```

The next CEO heartbeat picks up the new bundle automatically (cache invalidates on file content hash).
